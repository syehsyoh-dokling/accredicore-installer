# AccrediCore Installer

AccrediCore Installer is a production-oriented cross-platform launcher for preparing Windows, Linux, and macOS machines before deploying the main AccrediCore application.

The current Windows-first deployment flow is:

1. Check requirements
2. Install missing dependencies
3. Resolve port conflicts
4. Clone `accredicore-main` from GitHub
5. Create the local database and import the bundled schema
6. Import the customer-specific `.env` and `activation.json` files

## What is included

- Universal root launcher (`index.html`)
- Native desktop launcher using Electron
- Native runners for Windows, Linux, and macOS
- Requirement checkers that generate a shared JSON report format
- Dependency installers for each operating system
- Git bootstrap scripts
- Database bootstrap scripts
- Activation configuration import flow
- Build artifact flow for Windows, Linux, and macOS
- GitHub Actions workflow for artifact generation

## Local usage

### Browser-only mode
Open `index.html` in a browser. This is useful for previewing the UI and selecting the correct OS path manually.

### Desktop mode
Install Node.js 20+, then run:

```bash
npm install
npm run app
```

This opens a desktop shell that can invoke the native scripts directly.

### Windows flow notes

- Step 1 now validates `git`, `node`, `pnpm/npm`, `docker`, and `psql`.
- Step 3 explicitly resolves conflicts for ports used by the local AccrediCore stack.
- Step 5 imports the schema from the cloned repository under `database/bootstrap/`.
- Step 6 stores `.env` and `activation.json` under `config/env/` in the cloned project.

## Build artifacts

### Windows
```bash

```

### Linux
```bash
npm install
npm run dist:linux
```

### macOS
```bash
npm install
npm run dist:mac
```

Artifacts are written to `artifacts/`.

## Production notes

- Windows dependency installation requires elevated PowerShell.
- Linux dependency installation requires `sudo` or root.
- macOS dependency installation requires Homebrew.
- Docker installation usually requires a manual first launch after package installation.
- Code signing and notarization are not enabled in this starter production package and should be added before external distribution.
